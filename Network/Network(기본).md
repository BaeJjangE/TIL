# 네트워크

## 네트워크 구조(3가지)

```bash
# 네트워크 엣지
 - 랩탑, 데스크탑, 웹 서버 등이 위치함
 - 웹브라우저, 애플리케이션 등이 동작
 
# 네트워크 코어
 - 라우터가 위치
  - 라우터 : 패킷을 분산시켜주는 역할, 데이터를 받아주는 역할
  
# 링크
 - 라우터 사이에서의 연결 : 큰 케이블
 - 작은 링크에서는 모뎀선을 사용
 - LAN, 무선 링크(wifi 등)도 사용
```

>  패킷 : 비트들의 집합



## 네트워크 엣지 분류(3가지) - 모델

```bash
# end systems
 - 구동되는 프로그램
  - ex) web, email 등
  
# client/server model
 - 24시간 구동중인 서버와 클라이언트 사이에서 통신을 주고 받는 모델
  - ex) 웹 브라우저/서버, email 클라이언트/서버 등
  
# peer-peer model
 - 서버의 자원을 사용하여 통신하는 방식
  - ex) zoom, Skype, BitTorrent, discord 등
```



## 인터넷에서 제공해주는 통신 서비스(기본)

```bash
# TCP
 - 사용자에게 신뢰성 있는 통신을 제공
  - 메시지가 유실되지 않고 원본 데이터 그대로 송신
  - 순서를 지키면서 도착지까지 전달
 - 받는 대상에 맞춰 속도를 조절하며 송신
 - WEB, FTP 등에서 사용하는 방식
 - 우편에서 등기 시스템과 비슷
  - 비용이 듦
   - ex) 컴퓨팅 리소스(자원), 네트워킹 리소스(자원)

# UDP
 - 사용자에게 비신뢰성 통신을 제공(비연결지향)
 - 받는 입장을 고려하지 않고 보내는 방식
 - 하나도 못 받으면 속도가 빠르다는 게 의미가 없음
 - 전화에서 주로 사용
  - 음성 데이터 패킷이 몇 개 유실되도 알아차리기 힘듬
 - 실시간 방송, 중계에서도 사용
 - 우편에서 편지 시스템과 비슷
  - 100퍼센트 전달 가능성이 존재하지 않음
```



## 프로토콜

```bash
# 통신 규약
 - ex) 전화를 하면서 바로 본론을 꺼내는 것이 아닌 간단한 인사를 주고 받은 후 이야기를 꺼		 내는 것과 비슷한 느낌
- 실제 메세지를 주고 받기 위한 준비 동작
- 같은 프로토콜이어야 통신 가능
```



## 네트워크 코어

```bash
# 네트워크 중심에 위치한 라우터
# 라우터가 데이터를 전달하는 방식(2가지)
 - circuit-switching
  - 출발지에서 목적지까지 가는 길을 미리 예약해놓고 특정 사용자가 사용하도록 만든 방식
  - 예전 유선 전화망
 - packet-switching
  - 패킷 단위로 받아서 그 때 그 때마다 올바른 경로를 지정하여 전송하는 방식
  - 현재 라우터에서 사용하는 방식
```



## 패킷 스위칭

```bash
- 패킷의 앞부분이 도달해야 맨 앞 비트가 도착 후 대기, 후속 비트가 도착하여 패킷이 완전체가   된 후에 계속 진행하는 방식
 - 올바른 경로는 라우터가 결정
```

## 클라이언트 - 서버 아키텍쳐

```bash
- 서버는 웹 서버, 클라이언트는 웹 브라우저라고 생각
- 서버는 24시간 동작해야 하며, 고정된 아이피를 가지고 있어야 함
 - 인터넷 상에 존재하는 모든 컴퓨터들은 주소를 가지고 있어야 하는데 그것이 IP 주소
- 클라이언트는 IP 주소를 갖고 있되 고정되어 있지는 않아도 됨
- 시스템 내부에서 프로세스들 간의 통신이나 클라이언트 프로세스와 서버 프로세스 간의 통신에   큰 차이는 없음
 - 소켓 인터페이스를 만들어서 연결시킨 후 전달
 - 클라이언트 프로세스와 서버 프로세스 간에 통신을 하기 위해서는 소켓의 주소를 알아야 함
  - 소켓은 포괄적인 느낌
   - 소켓의 주소 역할 : IP 주소, 포트
   - ex) 구글에 접속하기 위해서 172.217.27.36:80

계층이란 개념은 하위 계층에서 상위 계층에게 서비스를 제공해주는 것
- 애플리케이션 계층은 트랜스포트 계층에서 사용하는 서비스를 받게 됨
```



## 애플리케이션 계층에서 트랜스포트 계층에게 원하는 사항

```bash
1. data integrity
- 내가 원하는 데이터가 유실되지 않고 온전하게 목적지까지 도달하면 좋겠다는 사항

2. timing
- 내가 보내는 데이터가 원하는 시간 내에 도착하면 좋겠다는 사항
ex) 음성 패킷이 도달하는 타이밍과 관련

3. throughput
- 내가 보내는 데이터가 적절한 용량으로 전달됐으면 좋겠다는 사항
ex) 영화를 다운로드받을 때 적절한 용량으로 다운로드 받는 것과 관련

4. security
- 내가 보내는 데이터가 안전하게 도착하면 좋겠다는 사항

트랜스포트 계층에서는 data integrity 만 제공
- TCP에서 제공해주며 UDP 해당 사항마저도 제공해주지 않음
```



## HTTP : HyperText Transfer Protocol

```bash
# HTTP : HyperText Transfer Protocol
- 하이퍼텍스트 전달 프로토콜
- 웹 브라우저에서 웹 서버에 접근할 때 웹 서버에서 클라이언트에 텍스트를 전달해주는 것
 - HTTP request
  - 클라이언트에서 원하는 하이퍼텍스트 파일 이름 요청
 - HTTP response
  - 응답하여 디스크를 읽고 파일 전송
```
